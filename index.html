What is Js

  • Javascript is a lightweight, cross-platform, object-oriented computer programming language

  • Javascript is one of the 3 core technologies of the web development

  • Javascript is commonly used as a part of webpages

  • Today, javascript can be used in different places :

    Client-side: Js is traditionnaly used only in the browser

    Server-side: Thanks to node.js we can use Js on server as well

Javascript parsers and engines

  • Javascript code is executed on the browser locally on the js engine of the browser

  • js can also be executed on server using node.js code but js first run on the browser

  • Js engine is a programme that execute js code

  • First the code is passed to parser : it will read the code line by line and check if synthax is correct. the parser know the rules how js has to be written. If mistakes is found it will throw an error and stop the execution.

  • If js code is correct then the parser will produce a data structure known as the Abstract Synthax tree. Which is then translated to machine code.

  • Only when the code is translated to machine code that the code is executed.

Primitive vs Object

  • Primitives are stored in the variable. When we pass the primitive variable in a function the function will create a new variable with the same value. It means the variable passed to the function can't be changed.

  • The variable which will get the new object only store the reference in memory where the whole object is recorded. When we pass an object to a function the new variable will get the reference of the object. It means the can be accessed and modified inside the function execution stack.

String in ES6

  • Template literals are enclosed by the back-tick (` `) (grave accent) character instead of double or single quotes. Template literals can contain placeholders. These are indicated by the dollar sign and curly braces (${expression}). The expressions in the placeholders and the text between them get passed to a function. The default function just concatenates the parts into a single string. If there is an expression preceding the template literal (tag here), this is called a "tagged template". In that case, the tag expression (usually a function) gets called with the processed template literal, which you can then manipulate before outputting. To escape a back-tick in a template literal, put a backslash \ before the back-tick.

Ignore case and find more than once

  • You can match both cases using what is called a flag. There are other flags but here you'll focus on the flag that ignores case - the i flag. You can use it by appending it to the regex. An example of using this flag is /ignorecase/i. This regex can match the strings "ignorecase", "igNoreCase", and "IgnoreCase"

  • To search or extract a pattern more than once, you can use the g flag.

match method

match a literal with different possibilities

  • This is powerful to search single strings, but it's limited to only one pattern. You can search for multiple patterns using the alternation or OR operator: |.

  • This operator matches patterns either before or after it. For example, if you wanted to match "yes" or "no", the regex you want is /yes|no/

Wild card

  • Sometimes you won't (or don't need to) know the exact characters in your patterns. Thinking of all words that match, say, a misspelling would take a long time. Luckily, you can save time using the wildcard character: .

  • The wildcard character . will match any one character. The wildcard is also called dot and period. You can use the wildcard character just like any other character in the regex. For example, if you wanted to match "hug", "huh", "hut", and "hum", you can use the regex /hu./ to match all four words.

Character class

  • You learned how to match literal patterns (/literal/) and wildcard character (/./). Those are the extremes of regular expressions, where one finds exact matches and the other matches everything. There are options that are a balance between the two extremes.

  • You can search for a literal pattern with some flexibility with character classes. Character classes allow you to define a group of characters you wish to match by placing them inside square ([ and ]) brackets.

  • For example, you want to match "bag", "big", and "bug" but not "bog". You can create the regex /b[aiu]g/ to do this. The [aiu] is the character class that will only match the characters "a", "i", or "u".

Use cases for event delegation

  • When we have an element with a lots of child elements that we are interested in. In this case, instead of adding an event listener to all of the child element we simply added one to the parent element, and we can determine on which child element the event was fired.

  • When we want an event handler attached to an element that is not yet in the DOM when our page is loaded

Event bubbling

  • Event bubbling : the event bubbles up inside the dom tree until it can be catch by the event handler. If a button is clicked, and define as target element the event object contains the target property. it means whent the event bubbles up all the parent element will know which child element where the event was fired first, also called target.

  • Event delegation : if the event bubbles up in the dom tree and if we know where the event was fired, then we can simply attached a event handler to a parent and wait to the event to bubbles up

  • Event.target : When the event bubbles up the event handler can access the target property of the event. event.target will return an html node in the DOM.
